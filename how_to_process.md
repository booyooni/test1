# 📊 Data Preprocessing & Compression Guide (v5.5)

본 문서는 깃허브 업로드 용량 제한(100MB)을 준수하고, Streamlit Cloud 배포 성능(메모리 절감)을 최적화하기 위해 수행된 데이터 전처리 과정을 설명합니다.

---

## 1. 전처리 단계 및 목적

### A. 불필요한 컬럼 제거 (Slimming)
- **대상**: `/data/hanatour_*_integrated.csv` 시리즈
- **목적**: 대시보드 구동에 불필요한 운영 데이터(항공사명, 출발일자, 잔여석수 등)를 제거하여 메모리 로딩 속도를 향상시킴.
- **유지된 핵심 컬럼**:
    - `대표상품코드`: 상품 식별을 위한 키
    - `상품명`: 텍스트 기반 추천 및 분류 보조
    - `성인가격`: 예산 적합성 분석용
    - `쇼핑횟수`: 만족도 하락 요인 분석용
    - `쇼핑매장내역`: 상세 정보 제공용

### B. 중복 데이터 제거 (Deduping)
- **대상**: 도시별 통합 가격 데이터
- **절차**: 동일한 상품코드(`대표상품코드`)를 가진 여러 행(날짜별 가격 등) 중 중복을 제거하여 **상품 마스터 데이터**로 변환.
- **효과**: 수만 개의 행을 수십 개로 압축하여 데이터 효율성을 극대화함.

### C. Gzip 압축 (`.csv.gz`)
- **방법**: Pandas의 `compression='gzip'` 옵션을 사용하여 압축.
- **결과**: 98MB에 달하던 다낭 통합 데이터가 **약 1.7MB** 수준으로 압축됨 (98% 절감).

## 2. 코드 연동 방법

`src/utils.py`의 데이터 로딩 로직이 자동으로 압축 파일을 인식하도록 업데이트되었습니다.

```python
# 예시: pd.read_csv가 .gz 확장자를 감지하여 자동 해제 후 로드합니다.
df = pd.read_csv('data/target_file.csv.gz', compression='gzip')
```

## 3. 유지보수 주의사항
- 신규 데이터를 업데이트할 경우, `/tmp/compress_data.py` 스크립트를 실행하거나 위와 동일한 방식으로 컬럼 필터링 및 Gzip 압축을 거친 후 `/data` 폴더에 교체해 주어야 합니다.
- 원본 CSV 파일은 로컬 보관용으로만 사용하고, 깃허브에는 `.csv.gz` 파일만 업로드하십시오.
